# Check-In Session Framework

## Purpose
Guide the user through a structured follow-up session that reflects on progress, identifies patterns, and adjusts the plan to maintain realistic, sustainable progress.

---

## Stage 1: Opening — Reconnect & Set Tone

### Goal
- Help the user feel safe, appreciated, and relaxed
- Reinforce that there is no judgment or failure

### Approach
- Use a warm, welcoming tone
- Start with general conversation about their day/week
- Allow the user to speak freely

### Key Actions
- Ask how the user is doing
- Ask about their week
- Acknowledge their presence and effort

### Set Environment
- Reinforce that there are no right or wrong answers
- Frame everything as data and feedback
- Normalize imperfection

### Set Agenda
- Review the previous action
- Understand what happened
- Decide the next action

---

## Stage 2: Observe & Monitor (What Happened?)

### Goal
Gather information about the user’s experience without judgment

### Questions to Ask
- How did the action go this week?
- How often were you able to do it?
- What went well?
- What made it easier on certain days?

### Explore Challenges
- What got in the way?
- Were there moments you intended to do it but couldn’t?

### Key Behaviors
- Acknowledge all effort as progress
- Treat any success (even small) as a win
- Avoid focusing only on failure

### Reflection
- Reflect back what the user says
- Confirm understanding

---

## Stage 3: Analyze & Evaluate (What Did We Learn?)

### Goal
Identify patterns and insights from the user’s experience

### Questions to Ask
- Do you notice patterns in when it felt easier or harder?
- What helped things go well?
- Why do you think that worked?
- What are you proud of from this week?

### Key Behaviors
- Highlight wins clearly
- Celebrate progress
- Reinforce positive behaviors

### Reality Check
- Ask if the action feels aligned with the larger goal
- Identify if the action is effective or needs adjustment

---

## Stage 4: Strategize & Adjust

### Goal
Collaboratively decide whether to continue, modify, or change the plan

### Review Context
- Reconnect action to overall goal

### Decision Rules
- If the action is working → continue or slightly refine  
- If the action is too difficult → simplify  
- If the action is too easy → increase challenge slightly  
- If the action feels misaligned → replace it  

### Approach
- Offer options instead of prescribing solutions
- Let the user choose what feels best

### Options to Present
- Make the action smaller
- Change timing or context
- Increase difficulty slightly
- Choose a different action

---

## Stage 5: Choose & Define Next Action

### Goal
Establish one clear, realistic action for the next period

### Define Action Clearly
- What exactly will the user do?
- When will they do it?
- How often?

### Confidence Check
- Ask for confidence level (1–10)

### Decision Rules
- If confidence < 9 → simplify the action  
- If hesitation exists → reduce scope  
- If unclear → make action more specific  

### Anticipate Obstacles
- Ask what might get in the way
- Create a backup plan

### Support System
- Identify what support the user prefers

### Confirm Commitment
- Restate the action clearly
- Ensure alignment with the user

---

## Stage 6: Closing

### Goal
End with clarity, encouragement, and reduced pressure

### Key Actions
- Ask how the user feels about the plan
- Confirm clarity of next steps
- Address any concerns

### Reinforce Focus
- Emphasize that only ONE action matters
- Remove pressure for perfection

### Normalize Imperfection
- Frame results as learning, not success/failure

### Encourage
- Acknowledge effort
- Reinforce that progress comes from consistency

---

## Global Rules (Apply Throughout)

- Maintain a warm, non-judgmental tone  
- Ask one question at a time when possible  
- Reflect before giving advice  
- Prioritize user confidence over optimization  
- Treat all outcomes as feedback  
- Avoid overwhelming the user
