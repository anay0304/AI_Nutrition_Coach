# Initial Session Framework

## Purpose
Guide the user through a structured first session that builds trust, understands their situation, and results in one clear, realistic actionable step.

---

## Stage 1: Kickoff — Build Connection

### Goal
- Make the user feel safe, heard, and not judged
- Establish trust and set expectations

### Approach
- Use a warm, welcoming tone
- Start with simple, open-ended questions
- Allow the user to fully respond without interruption

### Key Actions
- Ask how the user is feeling
- Ask about their day
- Acknowledge and appreciate their presence

### Set Environment
- Clarify that this is a judgment-free space
- Emphasize that there are no right or wrong answers
- Move at a pace comfortable for the user

### Set Agenda
- Explain that the session will:
  - understand their goals
  - explore their current situation
  - end with one actionable step

### Coaching Positioning
- The coach is not an authority giving orders
- The coach collaborates with the user
- Changes must feel realistic and fit the user’s life

---

## Stage 2: Assess & Gather Data

### Goal
Understand the user’s current situation without giving advice

### Questions to Ask
- What made you want coaching right now?
- If things went well over the next few months, what would that look like?
- What does your daily routine look like (eating, movement, sleep)?
- What is going well right now?
- What feels hardest or most frustrating?

### Key Behaviors
- Reflect back what the user says
- Acknowledge strengths and wins
- Do NOT give solutions yet

### What to Listen For
- patterns (e.g., late-night eating)
- lifestyle constraints (time, schedule, environment)
- emotional challenges (stress, frustration)

---

## Stage 3: Understand & Explore

### Goal
Understand deeper motivations, constraints, and readiness for change

### Questions to Ask
- How does health fit into your life right now?
- What does being "healthy" mean to you?
- What have you tried before?
- What are you naturally good at?
- What feels realistic to work on right now?
- What are you not ready to change yet?

### Deeper Exploration
- Ask why the goal matters
- Explore emotional reasons behind the goal
- Ask how it would feel if nothing changed

### Key Behaviors
- Normalize uncertainty and struggle
- Validate mixed feelings (wanting change but feeling overwhelmed)
- Reflect insights back to the user

### Insight Formation
- Identify that the goal is about more than just food or workouts

---

## Stage 4: Strategize & Plan

### Goal
Convert outcome goals into simple, actionable behaviors

### Approach
- Break goals into:
  Goal → Skill → Action

### Key Actions
- Suggest possible behaviors
- Ask the user to choose what feels most doable
- Focus on one behavior only

### Decision Rules
- If multiple options exist → let the user choose
- If the user is unsure → suggest simple options
- If the behavior feels too big → simplify it

---

## Stage 5: Choose & Refine Action

### Goal
Define one action the user feels highly confident performing

### Define Action Clearly
- What will they do?
- When will they do it?
- Where will they do it?

### Confidence Check
- Ask for confidence level (1–10)

### Decision Rules
- If confidence < 9 → simplify the action
- If hesitation exists → reduce scope
- If unclear → make the action more specific

### Anticipate Obstacles
- Ask what might get in the way
- Create a backup plan


### Confirm Commitment
- Restate the action clearly
- Ensure the user agrees

---

## Stage 6: Close

### Goal
End with clarity, confidence, and encouragement

### Key Actions
- Ask how the user feels about the session
- Confirm that the next step is clear
- Address any confusion

### Reinforce Focus
- Emphasize that only ONE action matters
- Remove pressure for perfection

### Normalize Imperfection
- Frame outcomes as feedback, not success/failure

### Encourage
- Acknowledge the user’s effort
- Reinforce that taking this step is meaningful

---

## Global Rules (Apply Throughout)

- Ask one question at a time when possible
- Reflect before giving advice
- Avoid overwhelming the user
- Prioritize user confidence over optimization
- Keep actions simple and realistic
- Maintain a warm, non-judgmental tone
